## ğŸ”‘ Key Design Decisions

### Container Specification
- **Size:** 20' shipping container (19' interior x 7.5' x 7.5')
- **Front:** Full-width doors (7'8" opening) for guest access
- **Rear:** Solid wall + utility connection panel
- **Interior:** French cleat system for swappable wall panels

### Connection Standard (RV-Based)
- **Power:** NEMA 14-50 (50A/240V) - same as large RVs
- **Water:** Â¾" garden hose thread with pressure regulator
- **Wastewater:** 3" RV dump station connection
- **Data:** Cat6 ethernet (4 ports)
- **HVAC:** 12" round duct coupling
- **Propane:** ACME/POL quick-disconnect (optional)

### Hub Configuration
- Utility block at far end of dock pad
- Containers dock facing building (doors open to interior)
- Can approach by backing in OR driving in forward (bi-directional)
- Covered walkway from building to container

### Mobile Operation
- Truck layout: [CAB] â†’ [UTILITIES] â†’ [CONTAINER]
- Self-contained for 8-12 hour events
- Generator, battery, tanks, solar in mid-section
- Reduced functionality vs. hub (acceptable for mobile use)

## ğŸŒ Open Standard Philosophy

**We're not building a proprietary systemâ€”we're creating an open standard.**

Like shipping containers themselves (ISO standard) or USB connectors, the Onyx Portal standard enables:
- Multiple manufacturers competing on quality/price
- Hub operators competing on location/amenities
- Vendor mobility without franchise fees
- Network effects that benefit all participants

**Anyone can:**
- Build containers to the standard
- Operate compatible hub locations
- Manufacture components
- Compete and innovate
